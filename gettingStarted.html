2.0 Getting Started

2.1.0 FastAi

When beginning the project, I had no prior knowledge of the fastAi library or even of the implementation of machine learning in general, therefore my first course of action was to work through the fastAi course provided by Jeremy Howard.

https://course.fast.ai/

This free online insight into the fastAi library also delves into the fundamentals of machine learning and the construction of neural networks in a range of contexts such as vision, tabular data, and NLP. Whilst the project itself does not delve to deeply into these concepts, a foundation understanding is vital in being able to appreciate and modify these systems and its value is not to be underestimated.

Source: https://docs.fast.ai/

The fundamental structure of fastAi relies on passing a dataset as a data frame to the data block API which acts as a DataLoader for the learner API. The learner API can then apply a passed architecture to train a model over a fixed number of epochs, where an epoch is one complete cycle through the input dataset.

For those new to machine learning much of this will appear foreign but below I have included an example of a data block and a learner from the second lesson of the fastAi course, to provide a breakdown of their general structure.

(Source: https://colab.research.google.com/github/fastai/fastbook/blob/master/02_production.ipynb#scrollTo=PLMLwIKf9are )

2.1.1 Data Block

(Source: https://colab.research.google.com/github/fastai/fastbook/blob/master/02_production.ipynb#scrollTo=PLMLwIKf9are )

Firstly, the independent and dependant variables are passed to the data block, this states that the API will take an image as an input and return a category in response. In this instance, a .JPEG image of a bear can be passed to the system and a string value, categorising the bear is returned.

Secondly, the data block requires information as to how to get the independent variables of the model. In this instance an array of file paths is passed and can be used to fetch the corresponding images.

Thirdly, the data block must split the dataset into a training set for training the model and a testing set for measuring the accuracy of the model. The recommended split, (valid_pct), is approximately 20% in a bid to prevent over fitting.

Fourthly, the data block requires a get_y function to associate the dependent variables with corresponding variables to derive relationships and generate accurate predictions.

Finally, the data block needs to carry out the relevant augmentations on the data so it is all equally sized and normalised to a standard form so the system can accurately recognise and process it. The learner cannot accept data items with varying sizes, so this is an essential step.

2.1.2 Learner

(Source: https://colab.research.google.com/github/fastai/fastbook/blob/master/02_production.ipynb#scrollTo=PLMLwIKf9are )

Firstly, the data block declared previously must be passed to a data loader alongside the file path to the dataset. This is then passed to the learner on declaration and acts as the training and testing sets for the model.

The learner also declares which architecture to employ. In this instance it uses resnet18, an 18-layer CNN, but this can be varied with different architectures such as xResnet etc.

Finally, the learner declares which metric to measure, this allows the user to have a quantifiable figure of how successful the model is. The accuracy of the model is declared as 1-error_rate. The accuracy can also be passed as a metric.

The learn.fine_tune method is then called to begin training the pretrained model, the parameter of the method is the number of epochs to run for so in this instance the model will run for 4 epochs.

(Source: https://colab.research.google.com/github/fastai/fastbook/blob/master/02_production.ipynb#scrollTo=PLMLwIKf9are )

Within machine learning, loss is declared as a measure of the systems inaccurate predictions, therefore by minimising loss, you can maximise accuracy. This concept is explored further within lesson 4 of the fastAi course which is available here, https://colab.research.google.com/github/fastai/fastbook/blob/master/04_mnist_basics.ipynb .

2.2.0 Datasets

Within machine learning, the dataset is the corner stone of any model as this is the information that all future predictions will be trained off. Datasets will contain the independent and dependent variables of the model and a suitable get_y function will be required to accumulate this information. The problem is that at the time of writing there is no standard labelling convention for machine learning. Some datasets contain separate metadata CSV files, others incorporate the labels within the file name and others even do both.

Source: https://www.kaggle.com/kinguistics/heartbeat-sounds?select=set_a.csv

Throughout the project I came across 3 key methods for accessing datasets, fastAi’s UNTAR method, the Bing search API and my personal preference the Kaggle API.

    After becoming familiar with the theory behind fastAi and its applications, it Is important to apply what has been learned practically to reinforce the skills that you have developed.

    The best way to do this is to edit the existing notebooks. Change parameters and note the impact, change the datasets, and see if the accuracy of the model changes or remains constant.
